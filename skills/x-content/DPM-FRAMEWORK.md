# DPM Framework — Content Psychology for @ST_Automation X-Posts

> Loaded by `SKILL.md` as MANDATORY READ-FIRST for every draft.
> Quelle: Şafaks Master Vault (NotebookLM, integriert 2026-05-12).
> Wer das Dopamin kontrolliert, kontrolliert die Reichweite.

## 1. KERN-PRINZIP — Dopamine Per Minute (DPM)

X wird in Millisekunden gewonnen oder verloren. Es reicht nicht zu informieren — der Leser muss biologisch belohnt werden.

**Floor:** mindestens 150 Dopamin-Einheiten pro Konsumminute. Jeder Satz muss den Drang auslösen, den nächsten zu lesen (Action-Reward-Zyklus).

**Mechanik:** Open Loops (ungelöste Spannungsbögen) durch den ganzen Post. Wer einen Loop öffnet ohne ihn zu schließen, hält den Leser fest. Wer alles im ersten Satz auflöst, verliert ihn.

## 2. HOOK-PSYCHOLOGIE — Pattern Interrupt

Der erste Satz hat eine einzige Aufgabe: das Daumen-Scrollen stoppen.

### Reverse 21 Questions
Nenne das Tool, die Technik, den Begriff NIE zuerst. Beschreibe die Wirkung, den Effekt, das Resultat.

| Falsch | DPM-tauglich |
|---|---|
| „Hier ist mein n8n-Tutorial." | „Ich habe eine digitale Brücke gebaut, die meine Lead-Gen erledigt, während ich schlafe. 0 Zeilen Code." |
| „Claude Code Workflow für KMU." | „Ein Solo-Berater. 4 Kunden. Kein Team. Keine 60h-Wochen." |
| „MCP Server Demo." | „Mein Notebook ersetzt drei Mitarbeiter. Ich zahle 50 € im Monat." |
| „Seedance 2.0 Anti-Patterns." | „Ich habe 40 Renderings gebraucht um zu kapieren, warum drei Wörter alles kaputt machen." |

### Instant Novelty
Erzeuge milde Verwirrung gepaart mit hohem Interesse. Schock, Konflikt, radikaler Perspektivwechsel.

Patterns:
- **Lügen-Taktik:** „Mein Verstand hat mich 6 Monate lang belogen." (kognitive Dissonanz)
- **Inverse Authority:** „Alle sagen X. Ich habe getestet was passiert wenn man Y macht."
- **Number Bait:** „Ich habe 47 KI-Tools getestet. 44 sind unbrauchbar."
- **Konflikt-Hook:** „Mein erster Kunde hat mich gefeuert. Sechs Monate später schickt er mir Empfehlungen."

### Wie man die Information Gap öffnet (nicht schließt)
- Setup beschreiben, Lösung andeuten, aber nicht nennen
- Konkretes Ergebnis nennen, aber Methode nicht
- Mistake bekennen, aber Korrektur nicht (erstmal)

## 3. INHALTS-FRAMEWORK — Cool Thing without Uncool Thing

Das stärkste Resonanz-Pattern.

**Formel:** „Ich zeige dir, wie du [ERSTREBENSWERTES ZIEL] erreichst, ohne [HASSENSWERTE AUFGABE] zu tun."

| Cool (verkauft) | Uncool (vermeidet) |
|---|---|
| Content auf Autopilot | Vor leerem Word-Dokument sitzen |
| 4 Kunden gleichzeitig betreuen | 60-Stunden-Wochen schieben |
| Inbound-Leads generieren | Cold-Calls oder Cold-DMs schreiben |
| KMU-Automatisierung verkaufen | IT-Beratung mit Klemmbrett |
| Multi-Agent-Stack | Personal einstellen und führen |
| Build-in-Public Revenue | Pitches an Investoren halten |
| AI-UGC-Reels in 8 Sprachen | OmniHuman + Veo-Hand-Workflow von Hand bauen |

Verkaufe NIE die Technik (Claude Code, MCP, n8n, Python). Verkaufe das Resultat ohne den Schmerz.

## 4. STRUKTUR + RHYTHMUS — Visual Addictivity

Text muss sich „glatt" anfühlen, aber genug Reibung erzeugen um hängen zu bleiben.

### Chunking
Kleine, verdauliche Häppchen. Niemals dichte Absätze. Visueller Atem zwischen den Aussagen.

### 12-Jährige-Regel
Schreibe so, dass ein durchschnittlich begabter 12-Jähriger alles versteht. Wenn ein Begriff Erklärung braucht — ersetzen oder umschreiben.

**Ausnahme @ST_Automation:** Tool-Namen (Claude Code, Seedance, MCP, Veo) sind ICP-Hooks, die bleiben. Aber drumherum kein Jargon. Kein „Inferenz-Pipeline" — sage „Der Workflow läuft so:".

### Satzlängen-Variation
Anti-AI-Stakkato. Mensch-Rhythmus = kurz, kurz, lang, kurz. Niemals 4 Fragmente hintereinander. Niemals 4 Schachtelsätze hintereinander.

**Test:** Lies den Post laut. Wenn er klingt wie ein Bot oder ein Stakkato-Vortrag → umschreiben.

### Visual Elements (auf X erlaubt)
- Fettschrift für Akzente (sparsam, max 2x pro Post)
- Emojis als Anker für Aufzählungen (✅ ❌ → 🧵 sparsam, max 3 pro Post)
- Zeilenumbrüche als Atemzeichen

### Verbot — Horizontale Trennlinien IN POSTS
- KEIN `---`
- KEIN `———`
- KEIN `═══`
- KEINE `----` als Trenner

Diese wirken wie Stoppschilder für das Gehirn und töten das Momentum.

> **Klarstellung:** Diese Regel gilt für X-POSTS. Interne MD-Dateien (dieser Skill, `SKILL.md`, `x-strategy.md`) dürfen `---` als Sektionstrenner nutzen — das ist Dokumenten-Struktur, nicht Content.

## 5. THREADS — Value Machine

| Tweet # | Aufgabe |
|---|---|
| 1 (Hook) | Pattern Interrupt + Information Gap. Niemals auflösen. |
| 2 | Gap vergrößern. Konkretes Detail nennen, Hauptmethode noch zurückhalten. |
| 3 bis N | Value liefern. Pro Tweet ein Aha-Moment oder eine Mini-Auflösung. |
| N+1 (Closer) | Cliffhanger oder „mehr davon" → Follow-CTA implizit, nicht plump. |

### Aha-Moment — der Thread-Multiplier
Liefere eine neue Logik, die das bisherige Scheitern des Lesers erklärt.

Beispiele:
- „Nicht das Tool hält dich auf, sondern deine Annahme darüber."
- „Du löst das falsche Problem. Hier ist das echte."
- „Ich dachte, ich brauche mehr Tools. Ich brauchte weniger."
- „Es ist nicht der Prompt. Es sind die Felder, die du auslässt."

Wer den Leser smarter macht („smarter in a remarkable way") wird geteilt.

## 6. KOMMENTARE + REPLIES — Secret Signals

Reply-Strategie ist Hauptachse in Phase 1 (siehe `x-strategy.md`). DPM gilt auch dort.

**Forbidden in Replies:**
- „Great!" / „This!" / „💯" / „🔥" allein
- Generische Zustimmung ohne Substanz
- Plain Promo („check meinen Account")

**Required in Replies (eines davon, idealerweise zwei):**
- **Hyper-spezifisches Detail** das beweist, dass man das Original gelesen hat
- **Binäre Frage** (Ja/Nein), die ein Micro-Commitment erzwingt
- **Eigenes konkretes Resultat** (Zahl, Workflow, Tool) das den Punkt erweitert oder kontert

**Pattern für First-Reply-Bangers (an Top-Targets, <15min):**
1. Hyper-spezifisch zustimmen oder höflich kontern
2. Eigene Mini-Story (1 Satz, mit Zahl)
3. Open Loop einbauen — „funktioniert auch für X, aber nur wenn Y"

## 7. WORD SHIT LIST — Sofort-Disqualifikation

Diese Wörter signalisieren „generische Werbung" und disqualifizieren den Post beim Lizard Brain. Auch nicht abgewandelt nutzen.

**Englisch:**
- bespoke
- elevate
- empower
- landscape (im Sinne von „the AI landscape")
- enhance
- transform
- in today's fast-paced world
- seamless
- cutting-edge
- revolutionary
- unleash
- robust (bereits aus früherer Liste)
- leverage (bereits aus früherer Liste)
- showcase (bereits aus früherer Liste)
- delve (bereits aus früherer Liste)
- unlock (außer in konkretem Tool-Kontext wie „unlocked feature")
- game-changer (bereits gesperrt)
- mind-blowing (bereits gesperrt)
- this changes everything (bereits gesperrt)

**Deutsch (für DE-Replies + DACH-Cluster):**
- maßgeschneidert
- revolutionär
- bahnbrechend
- nahtlos
- innovativ
- in der heutigen schnelllebigen Welt
- ganzheitlich
- holistisch
- synergetisch
- effizient (außer mit konkretem Beispiel)
- skalierbar (außer mit konkretem Beispiel)

**Test:** Würde ein Mensch in den Schützengräben das so sagen, oder ein Roboter mit Klemmbrett? Wenn zweiteres → ersetzen.

## 8. DPM-CHECK (Quality Gate)

Vor jedem Post — durch alle Checks:

- [ ] Hook ist ein Pattern Interrupt (Reverse 21, Instant Novelty, Lügen-Taktik)
- [ ] Tool / Technik wird NICHT in der ersten Zeile genannt
- [ ] Cool-without-Uncool-Formel ist im Text (explizit oder implizit)
- [ ] Open Loop in Hook oder Setup, geschlossen erst in Reply / Thread-Verlauf
- [ ] Variierte Satzlängen (laut-lesen-Test)
- [ ] Kein Wort aus der Word Shit List
- [ ] Keine `---` oder horizontalen Trenner im Text
- [ ] 12-Jähriger könnte alles verstehen (außer Tool-Namen, die sind ICP-Hooks)
- [ ] Mindestens ein konkretes Detail (Zahl, Tool, Workflow-Step, Zeit-Vergleich)
- [ ] Aha-Moment oder Information Gap am Ende, der zum Folgen / Reply einlädt

**Floor: 8/10 Check-Points bestanden.** Sonst Rewrite-Loop max 3x. Wenn nach 3x noch <8 → Post verwerfen, neuer Topic.

## 9. ANWENDUNG IM AGENT-WORKFLOW

**Vor jedem Standalone-Draft:**
1. Diese Datei lesen
2. Draft erstellen
3. DPM-Check durchgehen (Sektion 8)
4. Bei <8/10 → Rewrite-Loop max 3x
5. Wenn nach 3x noch <8/10 → Post verwerfen, neuer Topic

**Vor jedem Top-Target-Reply (First-Reply-Banger):**
1. Original-Post lesen
2. Hyper-spezifisches Detail oder Mini-Story raussuchen
3. Reply mit Open Loop bauen
4. DPM-Check Sektion 6 + Word-Shit-List

**Vor jedem Thread:**
1. Hook nach Pattern Interrupt bauen
2. Gap-Erweiterung in Tweet 2 planen
3. Pro Body-Tweet ein Aha-Moment oder Mini-Auflösung
4. Closer mit Cliffhanger schreiben
5. Komplett-Lesung laut
